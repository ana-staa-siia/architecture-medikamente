## Инструменты, способы и меры для обеспечения конфиденциальности данных

### По этапам потоков данных

| Этап                           | Что делаем                                    | Чем / как                                          |
|:-------------------------------|:----------------------------------------------|:---------------------------------------------------|
| **Сбор данных**                | Получаем согласие клиента на обработку        | Форма согласия на портале, хранение факта согласия |
| **Передача (клиент → сервер)** | Шифруем канал                                 | HTTPS (TLS)                                        |
| **Передача (сервер → сервер)** | Аутентифицируем сервисы друг перед другом     | mTLS или JWT с коротким сроком жизни               |
| **Хранение в БД**              | Шифруем чувствительные поля                   | AES-256, ключи хранятся отдельно (Vault)           |
| **Хранение файлов**            | Шифруем файлы с медкартами и сканами          | Шифрование на уровне хранилища или объектов        |
| **Доступ сотрудников**         | Даём доступ только по ролям                   | Роли: врач, ресепшен, бухгалтер, аналитик          |
| **Доступ к чужим данным**      | Врач видит только своих пациентов             | Привязка по ID пациента (ABAC)                     |
| **Аналитика / BI**             | Перед выгрузкой убираем прямые идентификаторы | Обезличивание (удаление ФИО, телефона)             |
| **Логирование**                | Записываем, кто и к каким данным обращался    | Логи API, логи БД, сбор в центральной системе      |
| **Мониторинг**                 | Отслеживаем подозрительные действия           | Алерты при массовом чтении или доступе без прав    |
| **Удаление**                   | Чистим данные по истечении срока хранения     | Автоматические задачи очистки                      |
| **Интеграция с лабораторией**  | Защищённый обмен через API                    | OAuth2, проверка сертификатов                      |

### Дополнительные меры

| Мера                       | Зачем                                                            |
|:---------------------------|:-----------------------------------------------------------------|
| **Тегирование данных**     | Помечаем, какие данные чувствительные, какой у них срок хранения |
| **Регулярные аудиты**      | Периодически проверяем, кто и как работает с данными             |
| **Обучение сотрудников**   | Люди — самый слабый элемент, их нужно учить                      |
| **DLP (защита от утечек)** | Контролируем, что уходит за пределы компании                     |
