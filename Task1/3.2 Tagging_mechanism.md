## Механизм тегирования данных

### Типы тегов

| Тег                    | Значение                                         |
|:-----------------------|:-------------------------------------------------|
| `pii`                  | Персональные данные (ФИО, телефон, email, адрес) |
| `sensitive_medical`    | Медицинская тайна (диагнозы, анализы)            |
| `payment`              | Платёжные данные                                 |
| `retention:5y`         | Срок хранения 5 лет                              |
| `retention:10y`        | Срок хранения 10 лет                             |
| `deletable_on_request` | Может быть удалено по требованию                 |
| `anonymized`           | Данные обезличены                                |
| `encrypted`            | Данные зашифрованы                               |

### Механизм тегирования

1. **Назначение тегов** — при создании записи или загрузке файла система автоматически или вручную
   присваивает теги в зависимости от типа данных.

2. **Хранение тегов** — теги хранятся вместе с данными (в метаданных таблицы, в атрибутах файла, в
   схеме сообщения).

3. **Использование тегов**:
    - API Gateway проверяет теги перед передачей данных
    - Система мониторинга отслеживает доступ к тегированным данным
    - Политики доступа используют теги для разрешения/запрета операций

4. **Контроль на основе тегов**:
    - Данные с тегом `pii` доступны только авторизованным ролям
    - Данные с тегом `sensitive_medical` доступны только лечащему врачу
    - Данные с тегом `deletable_on_request` подлежат удалению по запросу клиента
    - Данные с тегом `anonymized` могут использоваться для BI и ML
