#### ПРОБЛЕМНЫЕ ЗОНЫ

1. Хранилища без контроля:
    - File system (общий диск) — любой сотрудник с AD может читать.
    - Patients.xls, Journal-Doctor-FIO.xls — без шифрования.
    - Data file — произвольные файлы с медтайной.
    - RegistryByDate.xls — лабораторные данные.

2. Отсутствие разграничения доступа:
    - Нет RBAC/ABAC.
    - Администратор сети имеет доступ ко всему.

3. Небезопасные протоколы и дублирование:
    - TCP/IP + OLE (устаревший, без TLS).
    - Двойной ввод данных (кассир вручную в Excel).

4. Нарушение Privacy by Design:
    - Данные собираются и хранятся без минимизации.
    - Нет механизма удаления по требованию.

5. Отсутствие аудита и логирования:
    - Неизвестно, кто и когда читал медкарты.

6. Неучтённые категории конфиденциальных данных:
    - Согласия на обработку ПДн не выделены как отдельный объект учёта.
    - Технические идентификаторы (IP, session id, device id) не классифицированы как чувствительные
      метаданные.
    - Нет явного владельца данных (data owner) по ключевым наборам.
