## Аудит мер безопасности

### Сопоставление процессов с требованиями по безопасности

| Процесс                                 | Что должно быть                              | Что есть сейчас                              |
|:----------------------------------------|:---------------------------------------------|:---------------------------------------------|
| Запись пациента (администратор → файлы) | Передача и хранение ПДн должны быть защищены | Данные передаются и хранятся в открытом виде |
| Хранение медкарт (Data file)            | Доступ только по ролям                       | Общий диск, доступный всем сотрудникам       |
| Передача ККМ → 1С                       | Безопасный канал связи                       | TCP/IP + OLE, без шифрования                 |
| Двойной учёт кассира (Excel + 1С)       | Хранить данные в одном месте                 | Избыточное копирование ПДн                   |
| Доступ к Laboratory Registry            | Нужно знать, кто и когда обращался к данным  | Логирование отсутствует                      |
| Удаление данных по запросу клиента      | Должна быть возможность удалить данные       | Не реализовано                               |
| Интеграция с лабораторией               | Автоматизированный защищённый обмен          | Передача на бумаге или по PDF вручную        |

### Сопоставление с архитектурными практиками

| Практика                | Что означает                                       | Реализовано?                 |
|:------------------------|:---------------------------------------------------|:-----------------------------|
| Privacy by Design       | Защита закладывается в архитектуру с самого начала | Нет                          |
| Data Minimization       | Собираются только необходимые данные               | Нет (хранится лишнее)        |
| Data Lineage            | Отслеживается происхождение и изменения данных     | Нет                          |
| Zero Trust              | Доступ не доверяется никому по умолчанию           | Нет (AD даёт широкий доступ) |
| RBAC / ABAC             | Разграничение доступа по ролям или атрибутам       | Нет                          |
| Шифрование при хранении | Данные на диске зашифрованы                        | Нет                          |
| Шифрование при передаче | Данные в сети зашифрованы                          | Нет                          |
| Аудит доступа           | Логируются все действия с данными                  | Нет                          |
| Управление согласиями   | Есть механизм сбора и отзыва согласий              | Нет                          |

### Итог

Большинство базовых мер безопасности отсутствует. Требуется перестройка архитектуры с приоритетом на
защиту данных.
