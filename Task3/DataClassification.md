## Оценка Data Encryption at Rest and In Transit

| Данные                                                      | В чём необходимость защиты                                            | Инструменты защиты                                                                                                                                                       |
|:------------------------------------------------------------|:----------------------------------------------------------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ПДн пациента (ФИО, телефон, email, дата рождения, адрес)    | Требования законодательства РФ, риск утечки и мошенничества           | **In Transit**: TLS 1.3; **At Rest**: AES-256; **Access**: RBAC (только нужные роли), тегирование (pii); **Keys**: Vault/KMS                                             |
| Медицинские данные (диагноз, анализы, медкарта, назначения) | Медицинская тайна, высокий репутационный и финансовый риск            | **In Transit**: mTLS между сервисами; **At Rest**: шифрование полей БД и файлов; **Access**: ABAC (только свои пациенты), тегирование (sensitive_medical), аудит доступа |
| Платежные данные (сумма, чек, токен платежа)                | Риск фрода и несанкционированных транзакций                           | **In Transit**: TLS; **At Rest**: маскирование (последние 4 цифры), токенизация; **Access**: API Gateway с проверкой прав                                                |
| Документы и сканы (договоры, согласия, PDF/JPG)             | Содержат ПДн и юридически значимые сведения                           | **In Transit**: TLS; **At Rest**: шифрование файлов (AES-256); **Access**: RBAC, тегирование (pii, deletable_on_request)                                                 |
| Учетные данные (логины, хэши паролей, токены)               | Компрометация дает доступ к чувствительным данным системы             | **At Rest**: bcrypt/Argon2, секреты в Vault; In Transit: TLS; **Access**: короткоживущие JWT, MFA                                                                        |
| Логи доступа и аудита                                       | Нужны для расследования инцидентов и подтверждения соблюдения политик | **At Rest**: WORM-хранилище (неизменяемость), шифрование; **Monitoring**: SIEM + Victoria Metrics, алерты на подозрительные действия                                     |
| Обезличенные данные для BI/ML/AI                            | Риск повторной деанонимизации, нужен контроль использования           | **In Transit**: TLS; **At Rest**: шифрование; **Access**: только через Data Lake, тегирование (anonymized), политики OPA, нет прямых идентификаторов                     |
| Согласия на обработку ПДн                                   | Юридическое требование, нужно доказать факт согласия	                 | **At Rest**: неизменяемое хранилище, шифрование; **Access**: только аудиторы; **Process**: Consent Management Service                                                    |
